本文记录了我完成“利用云平台搭建个人博客”实验的全过程，并按照实验要求将实验报告整理为一篇博客文章发布。

## 实验目的

通过实践操作，掌握基于云计算环境的网站部署技能。具体包括：完成个人博客的搭建、掌握静态网站生成与发布流程、理解代码托管与自动部署机制，并将实验报告以博文形式发布到个人网站上。

## 实验环境

- 硬件环境：联网计算机一台
- 软件环境：Windows、Git、Node.js、Hexo、GitHub、GitHub Pages、GitHub Actions

## 实验步骤与内容

### 1. 准备开发环境

首先在本机安装 Git 与 Node.js，保证可以正常执行 `git`、`npm` 等命令。随后创建远程 GitHub 仓库：

- 仓库地址：`https://github.com/wang0-sudo/blog`
- 博客地址：`https://wang0-sudo.github.io/blog/`

### 2. 初始化 Hexo 博客

在本地使用 Hexo 初始化博客项目：

```bash
npm install -g hexo-cli
hexo init myblog
cd myblog
npm install
```

初始化完成后，生成了 `source`、`themes`、`public` 等目录，其中 `source/_posts` 用于编写文章，`public` 用于存放静态页面生成结果。

### 3. 配置博客站点

为了使 GitHub Pages 可以正确访问博客，需要在 `_config.yml` 中配置站点地址：

```yaml
url: https://wang0-sudo.github.io/blog
root: /blog/
```

这样 Hexo 在生成页面时，会自动拼接正确的静态资源路径和页面链接。

### 4. 撰写实验报告文章

根据实验要求，我将实验报告直接写成一篇 Markdown 博文，存放在 `source/_posts` 目录下。这样既满足了实验报告撰写要求，也实现了“将报告作为博文发布”的任务目标。

### 5. 配置自动部署

本项目使用 GitHub Actions 进行自动构建和发布。仓库中的 `.github/workflows/pages.yml` 配置了以下流程：

1. 在 `main` 分支有新提交时触发工作流。
2. 自动安装项目依赖。
3. 执行 `npm run build` 生成 `public` 静态页面。
4. 将构建结果上传并发布到 GitHub Pages。

这种部署方式不需要手动上传网页文件，体现了云平台自动化部署的优势。

### 6. 本地测试与远程发布

在提交之前，可以先本地预览博客效果：

```bash
hexo server
```

确认文章页面显示正常后，执行：

```bash
npm run build
git add .
git commit -m "add experiment report post"
git push -u origin main
```

推送成功后，GitHub 会自动构建并发布站点。

## 实验结果

通过本次实验，我成功搭建了基于 Hexo 的个人博客，并借助 GitHub Pages 将博客部署到云端，实现了在线访问。同时，实验报告也已经整理为博客文章，满足了实验要求。

## 结论分析与体会

本次实验让我对云平台环境下的网站部署流程有了更清晰的认识。相比传统部署方式，GitHub Pages 配合 GitHub Actions 的方案更加轻量、方便，也更适合个人博客和静态网站的开发与维护。

在实践过程中，我进一步掌握了 Hexo 博客框架、Git 版本控制以及自动化部署的基本流程，也深刻体会到云平台在资源管理、服务发布和持续集成方面的优势。通过这次实验，我不仅完成了博客搭建任务，也提升了独立部署和维护个人网站的能力。
